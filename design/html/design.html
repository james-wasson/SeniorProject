  1. Introduction

This document will provide the planned implementation for the Inventor Access Portal project. Its intended audience is the project manager, project team, and development team.

Below are the major portions described in this document:

  1. The User Interfaces’ appearance, functionality, and ease of use.
  2. The layout of the User Interface as well as how the information from the database will be received.
  3. The Architectures being used for the software and for security in this project. Each of these will be a key component in the software and should be paid close attention to.
  4. A number of Use Cases will be given so as to provide a benchmark for the project and some Verification that our project is including the most fundamental features.
  5. Finally an overview of Database Design, it will cover the database architecture as well as how the database shall be updated from the OTT’s access databases.

  1. GENERAL OVERVIEW AND DESIGN GUIDELINES/APPROACH

  1. Assumptions / Constraints / Standards

Assumptions

  • The database shall be stored by OTT or through some arrangement with a third party.
  • The OTT will update all other information pertaining to the user when changes should occur to their files or projects.

Constraints

  • The only fields that we can update are the password, username, suspended status, and temporary passwords, meaning the only thing the group can change is the login table information.
  • The color palette of the portal must also match that of the Office of Technology Transfer website.

Standards

  • The main architecture being used for this project is an ASP.Net Model-View-Controller (MVC) architecture. We’ll also be using Razor to dynamically generate HTML.
  • We will be using Microsoft SQL Server Migration Assistant (SSMA) for Access to transfer the data from the three access databases into our SQL server.

  1. Guiding Design Principles

  • One of the major focuses of this project is modularity in order to keep the project as flexible and independent of static technologies where possible. That is why this project uses a major architecture (MVC) that has a long history of software design and is backed by standard setters such as Microsoft.
  • The other major focus is security. Taking into special consideration the transfer of data from one database to the other using an encrypted VPN connection. Enforcing a HTTPS connection, which is the current security standard and applying anti-forgery tokens where necessary (i.e. Login) to avoid a man-in-the-middle attack. We also use Microsoft's Identity to secure the login data of the user and prevent unauthorized access to web pages.

  1. ARCHITECTURE DESIGN

  1. Software Architecture

The software architecture being used for this project is an ASP.Net Model-View-Controller (MVC) architecture.

  •  The view is what the inventor will see upon login, as the view controls what is displayed.
  •  The model gives the general format for the data presented in the view.
  •  The controller uses the data defined in the model and updates the view with that information.

The software shall use Entity framework with a model generated from a merged SQL database as described in this document: https://msdn.microsoft.com/en-us/library/jj206878(v=vs.113).aspx

  1. Security Architecture

  • The project shall implement Microsoft’s Forms Authentication along with anti-forgery tokens to validate a user when they login the website will secure their username and password.
  • The website will ensure HTTPS; which is the current security standard; and we will also be using Transport Layer Security (TLS) 1.2 which is the latest version and should have the highest level of security.
  • The project shall also ensure no unauthorized access to web pages using Forms Authentication and page filters as the first line of defense.
  • The project shall protect against SQL injection, testing inputs and setting max limits on all data inputs that are stored in the database.
  • Password storage shall use a randomly generated salt appended to each password and stored in the database to ensure password strength during hashing.

  1. Performance

Performance should be maximized in this project because the SQL server is local so access time; which is the biggest portion of the wait normally; should be very small.

  1. User Interface Design

  1. The first screen presented to the user will be the login screen where the user inputs their username and password.
  2. Upon signing in, before the user can continue they must agree to a confidentiality agreement.
  3. Pressing the button on the confidentiality notice will then send the user to the ‘Home’ Page which will consist of a table titled “Recent Activities” (shows all activities from most to least recent) which will be populated with data from the database. These entries will contain information such as date, activity, details, notes, file number and name, status, serial number, and law firm. The user will be able to sort entries by clicking on the columns.
  4. There will be a navigation bar to the left that will take the user to other tables where data is stored, beyond Recent Activities there will be a page for All Inventions relevant to the user, All Files, and any Project Families the user is a part of. The navigation bar will be similar to the one here.
  5. The All Inventions table will be sortable by status, project number, and project title.
  6. The All Files table will be sortable by status, file number, file name, serial number, continuity, law firm, and project number.
  7. The All Families table will be sortable by status, family number, and family name.
  8. Clicking on a given entry in any of these tables will send the user to a report form that pertains to whichever table they selected the entry from, i.e. selecting an entry in the All Files table will send the user to a file report form for that file.
  9. The user will then easily be able to navigate out of the report page by clicking a back button or using the navigation bar.
  10. The general appearance of the portal will match the appearance of the Office of Technology Transfer website.
